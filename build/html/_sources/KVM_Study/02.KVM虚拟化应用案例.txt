KVM虚拟化应用案例
=================

KVM虚拟化的核心主要由以下两个模块组成：

::

   1）KVM内核模块，它属于标准Linux内核的一部分，是一个专门提供虚拟化功能的
   模块，主要负责CPU和内存的虚拟化，包括：客户机的创建、虚拟内存的分配、CPU执行
   模式的切换、vCPU寄存器的访问、vCPU的执行。


   2）QEMU用户态工具，它是一个普通的Linux进程，为客户机提供设备模拟的功能，
   包括模拟BIOS、PCI/PCIE总线、磁盘、网卡、显卡、声卡、键盘、鼠标等。同时它通过
   ioctl系统调用与内核态的KVM模块进行交互。

在KVM虚拟化架构下，每个客户机就是一个QEMU进程，在一个宿主机上
有多少个虚拟机就会有多少个QEMU进程；客户机中的每一个虚拟CPU对应QEMU进程中的一个执行线程；一个宿主机中只有一个KVM内核模块，所有客户机都与这个内核模块
进行交互。

KVM上层管理工具
---------------

1.libvirt

::

   libvirt是使用最广泛的对KVM虚拟化进行管理的工具和应用程序接口，已经是事实上
   的虚拟化接口标准.

2.virsh

::

   virsh是一个常用的管理KVM虚拟化的命令行工具，对于系统管理员在单个宿主机上
   进行运维操作，virsh命令行可能是最佳选择。

3.virt-manager

::

   的。virt-manager除了提供虚拟机生命周期（包括：创建、启动、停止、打快照、动态迁移等）管理的基本功能，还提供性能和资源使用率的监控，同时内置
   了VNC和SPICE客户端，方便图形化连接到虚拟客户机中。

4.Openstack

::

   OpenStack仍然使用libvirt API来完成对底层虚拟化的管理

安装KVM组件前提
---------------

::

   安装KVM 组件需要先确保系统己经满足了最低安装要求,部署KVM 虚拟化需要通过BIOS
   开启CPU 的虚拟化功能,确保至少6GB 的磁盘剩余空间以及2GB 内存空间。
   KVM 虚拟机支持的存储方式有:本地磁盘文件、物理磁盘分区、LVM 分区、iSCS1 磁盘、GFS2 文件系统、光
   纤LUNs 设备等。

在部署操作系统时安装KVM组件
~~~~~~~~~~~~~~~~~~~~~~~~~~~

::

   KVM 虚拟化组件可以在安装操作系统的过程中选择《带GUI 的服务器》， 然后点选如下这
   些软件组包Virtualization CIient 、Virtualization Hypervisor 和VirtuaIization Tool来部署虚拟化软件环境。

通过Kickstart方式安装系统时需要在Kickstart配置文件中加入如下内容：

::

       @virtualization—hypervisor
       @virtualization—client
       @virtualization—platform
       @virtualization—tools

操作过程
--------

KVM虚拟机配置文件位置

::

   [root@bogon ~]# ll /etc/libvirt/qemu/centos7.xml 
   -rw------- 1 root root 4076 5月  22 18:11 /etc/libvirt/qemu/centos7.xml

修改KVM虚拟机配置的方法

::

   [root@bogon ~]# virsh edit centos7
   使用该命令修改可以对文件进行语法校验。

备份与恢复
~~~~~~~~~~

   [root@bogon ~]# virsh dumpxml centos7 > centos7.xml

导入虚拟机
~~~~~~~~~~

导入

::

   [root@bogon ~]# virsh define centos7.xml 
   定义域 centos7（从 centos7.xml）

查看

::

   [root@bogon ~]# virsh list --all
    Id    名称                         状态
   ----------------------------------------------------
    -     centos7                        关闭
    

修改虚拟机名称
~~~~~~~~~~~~~~

::

   [root@bogon ~]# virsh domrename centos7 centos7_hu
   Domain successfully renamed

   [root@bogon ~]# virsh list --all
    Id    名称                         状态
   ----------------------------------------------------
    -     centos7_hu                     关闭

虚拟机挂起与恢复
~~~~~~~~~~~~~~~~

开启虚拟机

::

   [root@bogon ~]# virsh start centos7_hu
   域 centos7_hu 已开始

   [root@bogon ~]# virsh list --all
    Id    名称                         状态
   ----------------------------------------------------
    1     centos7_hu                     running

虚拟机设置为开机自启动

::

   [root@bogon ~]# virsh autostart centos7_hu
   域 centos7_hu标记为自动开始

   # 实质为创建软连接
   [root@bogon ~]# ll /etc/libvirt/qemu/autostart/centos7_hu.xml 
   lrwxrwxrwx 1 root root 32 5月  23 10:09 /etc/libvirt/qemu/autostart/centos7_hu.xml -> /etc/libvirt/qemu/centos7_hu.xml

挂起虚拟机

::

   [root@bogon ~]# virsh suspend centos7_hu
   域 centos7_hu 被挂起

   [root@bogon ~]# virsh list --all
    Id    名称                         状态
   ----------------------------------------------------
    1     centos7_hu                     暂停

恢复虚拟机

::

   [root@bogon ~]# virsh resume centos7_hu
   域 centos7_hu 被重新恢复

   [root@bogon ~]# virsh list --all
    Id    名称                         状态
   ----------------------------------------------------
    1     centos7_hu                     running

查询虚拟机VNC端口
~~~~~~~~~~~~~~~~~

::

   [root@bogon ~]# virsh vncdisplay centos7_hu
   :0
   # :0 即 为 5900 端口，以此类推 :1为5901 。

开机自启动设置
~~~~~~~~~~~~~~

::

   # 设置 libvirtd 服务开机自启动。
   [root@bogon ~]# systemctl is-enabled libvirtd.service 
   enabled

案例
~~~~

①查看当前正在运行的虚拟机列表和所有虚拟机列表

::

   [root@bogon ~]# virsh list
    Id    名称                         状态
   ----------------------------------------------------
    1     centos7_hu                     running

   [root@bogon ~]# virsh list --all
    Id    名称                         状态
   ----------------------------------------------------
    1     centos7_hu                     running

② 设置虚拟机centos7_hu设置为开机自启动

::

   [root@bogon ~]# virsh autostart centos7_hu
   域 centos7_hu标记为自动开始

③ 备份centos7_hu虚拟主机的配置文件，将配置文件保存为centos7_clone.xml

::

   [root@bogon ~]# virsh dumpxml centos7_hu > centos7_clone.xml
   [root@bogon ~]# ll
   总用量 56
   -rw-------. 1 root root  1863 2月  13 21:54 anaconda-ks.cfg
   -rw-r--r--  1 root root  5151 5月  23 10:19 centos7_clone.xml

④
使用centos7_clone.xml配置文件生成新的虚拟主机，修改uuid、虚拟主机名称、MAC地址等必须修改的内容

::

   #随机生成UUID
   [root@bogon ~]# uuidgen
   4a7d0ddf-0211-4c69-8fc7-b6a03ed4c697

   < name > < / name > 定义的是虚拟主机名称， < uuid > < / uuid > 定义的是虚拟主机唯一的序列编号，
   <mac address ／ > 定义的是虚拟主机的MAC 地址。以上这些信息都是必须要修改的内容

|image6|

::

   <interface type='network'>
     <mac address='52:56:00:3e:29:57'/>
     <source network='default' bridge='virbr0'/>
     
   <graphics type='vnc' port='5901' autoport='no' listen='0.0.0.0'>
       <listen type='address' address='0.0.0.0'/>
       
       
       修改xml配置文件
   　　1）name
   　　2）uuid
   　　3）虚拟磁盘存储路径
   　　4）mac地址
       
   [root@bogon ~]# virsh create centos7_hu3_clone.xml 
   域 centos7_hu3 被创建（从 centos7_hu3_clone.xml）

⑤ 查看虚拟机centos7_hu3的基本信息

::

   #查看虚拟机的基本信息

   [root@bogon ~]# virsh list --all
    Id    名称                         状态

    1     centos7_hu                     running
    4     centos7_hu3                    running

   [root@bogon ~]# virsh dominfo 4
   Id:             4
   名称：       centos7_hu3
   UUID:           4a7d0ddf-0211-4c69-8fc7-b6a03ed4c697

   持久：       否

   安全性 DOI： 0

⑥ 开启centos7_hu3这台虚拟主机

::

   [root@bogon ~]# virsh start centos7_hu3
   错误：域已经活跃

   [root@bogon ~]# virsh list
    Id    名称                         状态
   ----------------------------------------------------
    1     centos7_hu                     running
    4     centos7_hu3                    running

⑦ 设置指定虚拟机内存大小值、最大内存值、CPU个数，使虚拟机下次重启后生效

::

   [root@bogon ~]# virsh dominfo 1
   Id:             1
   名称：       centos7_hu
   UUID:           30c7f379-636e-4911-b478-721bc7f51d5e
   OS 类型：    hvm
   状态：       running
   CPU：          1
   CPU 时间：   32.8s
   最大内存： 1048576 KiB
   使用的内存： 1048576 KiB
   持久：       是
   自动启动： 启用
   管理的保存： 否
   安全性模式： none
   安全性 DOI： 0

   # 调整最大内存
   [root@bogon ~]#  virsh setmaxmem centos7_hu 1048574 --config  

   #从1颗cpu增至2颗
   [root@bogon ~]# virsh setvcpus centos7_hu --maximum 2 --config

   #设置指定虚拟机内存大小值
   [root@bogon ~]# virsh setmem centos7_hu 1048572 --config

   [root@bogon ~]# virsh net-dumpxml default > default_net.xml

   -rw-r--r--  1 root root  5152 5月  23 10:31 centos7_hu3_clone.xml
   -rw-r--r--  1 root root   436 5月  23 10:47 default_net.xml
